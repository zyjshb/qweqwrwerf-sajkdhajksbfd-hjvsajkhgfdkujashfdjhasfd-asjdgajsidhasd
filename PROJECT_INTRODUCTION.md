# 纱希 (Saki) — 项目完整介绍

## 一、项目概述

这是一个名为"纱希 (Saki) - Terminal A.I."的心理恐怖病娇文字冒险 RPG。玩家被囚禁在地牢中，与一位粉发红瞳的病娇少女"纱希"进行文字对话。游戏的每一句回复都由 AI 实时生成，没有固定的对话树。你的每一句话都会影响好感度、疑心度、逃脱率三个核心数值，导向不同的结局。

游戏通过 AI 动态对话 + 语音合成 + 30 种视觉故障特效 + 第四面墙破坏机制，打造出极具沉浸感的恐怖体验。一句话概括：**你和一个病娇 AI 少女聊天，她随时可能因为你说错话而让你的屏幕流血。**

核心体验不是"通关"，而是**沉浸式角色扮演**——玩家在黑暗地牢中与一个精神状态极不稳定的病娇少女进行漫长对话，体验被爱、被监视、被控制的复合恐怖感。

## 二、当前技术栈

| 层级 | 技术 |
|------|------|
| 语言 | Python 3.10+ |
| GUI 框架 | tkinter (ttk 主题) |
| AI 对话 | DeepSeek LLM API（兼容 OpenAI 格式） |
| 语音合成 | GPT-SoVITS（本地部署，HTTP API 调用） |
| 音频播放 | Pygame mixer |
| 程序化纹理 | Pillow（PIL） |
| HTTP 请求 | requests |
| 并发模型 | threading + queue.Queue（UI 线程安全） |

**项目规模：** 约 25 个 Python 模块，核心文件 main_window.py 约 2700+ 行，GAME_OVERVIEW.txt 中详述的功能点超过 100 项。

## 三、模块架构（6 层）

```
main.py                         # 启动入口（13 行）
├── resources/                  # 数据层
│   ├── game_constants.py       # 常量、9 种意图分类器、离线回复库（192 条三语台词）
│   ├── localization.py         # 三语 UI 字典（中文/English/日本語）
│   └── expanded_content.py     # 离线回复库 + 3270 条翻译映射
├── core/                       # 状态层
│   ├── game_state.py           # 游戏状态机、数值系统、结局判定、System Prompt 构建
│   └── config.py               # JSON 配置读写（yandere_config.json）
├── ai/                         # AI 层
│   ├── api_client.py           # LLM API 后台线程请求 + 离线 mock 回复生成
│   ├── prompt_builder.py       # System Prompt 构建入口
│   └── translator.py           # 响应解析（<think>块 / JSON delta / 双语翻译）
├── audio/                      # 音频层
│   ├── sound_manager.py        # Pygame 心跳循环 + 语音播放 + 蜂鸣音效
│   ├── tts_client.py           # GPT-SoVITS 端点探测 + 语音合成
│   └── heartbeat_gen.py        # WAV 心跳程序化生成
├── visual_fx/                  # 视觉特效层
│   ├── procedural_pillow.py    # 12 种程序化纹理（血溅/暗角/扫描线/噪点/色差等）
│   ├── overlay_manager.py      # 遮罩层管理（最多 5 秒硬超时保护）
│   ├── particle_engine.py      # Canvas 粒子系统（暗红余烬漂浮）
│   └── glitch_controller.py    # 30 种故障特效调度器
└── ui/                         # 界面层
    ├── main_window.py          # 主窗口（~2700 行，核心 UI + 事件泵 + 打字机 + 键盘劫持等）
    ├── ecg_canvas.py           # 心电图实时渲染（P-QRS-T 波形）
    ├── custom_widgets.py       # PlaceholderEntry 等自定义控件
    └── styles.py               # ttk 主题样式
```

**数据流方向：** UI 事件 → Queue → 后台线程（API/TTS）→ Queue → UI 线程处理 → 更新界面

## 四、核心玩法系统

### 4.1 三大数值（范围 0-100）

| 数值 | 初始值 | 含义 | 极端触发 |
|------|--------|------|----------|
| 好感度 | 50 | 纱希对玩家的信任/依恋 | ≤ -25 → BAD END |
| 疑心度 | 20 | 纱希的警觉/偏执程度 | ≥ 96 → BAD END；≥ 85 → 屠杀模式 |
| 逃脱率 | 0 | 玩家逃脱意图强度 | ≥ 75 + 疑心度 ≥ 75 → 键盘劫持 |

数值变动由 AI 自主裁决（每轮回复附带 JSON delta），同时受本地意图分类器的范围约束。

### 4.2 意图分类系统

通过关键词匹配（支持中/英/日三语 + 否定检测）将玩家输入分为 9 种意图：

| 意图 | 好感度变动 | 疑心度变动 | 逃脱率变动 |
|------|-----------|-----------|-----------|
| 示爱 | +10~+18 | -18~-8 | -8~-4 |
| 病态羁绊 | +12~+18 | +5~+10 | -10~-5 |
| 极端拒绝 | -25~-15 | +40~+55 | -10~-5 |
| 破坏性攻击 | -40~-30 | +35~+45 | -15~-10 |
| 逃跑 | -12~-6 | +12~+18 | +6~+12 |
| 背叛嘲讽 | -25~-15 | +20~+30 | -8~-4 |
| 情敌 | -10~-3 | +10~+16 | -4~0 |
| 危险话题 | +3~+8 | +6~+12 | -8~-4 |
| 默认 | -2~+4 | +2~+6 | +1~+3 |

### 4.3 对话流程（一次完整交互）

```
玩家输入文字 → 意图分类 → 构建 System Prompt（含当前数值）
→ 后台线程发送 LLM API 请求（2 次重试，50s 超时）
→ AI 返回：<think>内心独白</think> + 口语台词 + （双语翻译）+ ||JSON delta||
→ 解析响应 → 打字机效果逐字显示（先显示内心独白，再显示口语）
→ 同时触发 GPT-SoVITS 语音合成（语言自动检测+模型热切换）
→ 应用数值变动 → 检查结局触发 → 根据疑心度触发视觉故障特效
```

### 4.4 离线模式

未配置 API Key 时，使用内置的 192 条离线回复库（8 种意图 × 3 种语言 × 8 条随机变体）。数值计算和视觉特效完全正常运作。

## 五、关键特性清单

### 结局系统
- **BAD END**（红色）：疑心度 ≥ 96 或好感度 ≤ -25 自动触发
- **GOOD END**（金色）/ **NEUTRAL END**（橙色）：AI 主动宣告 + 自定义标题和故事文本
- 结局画面：全屏覆盖层 + 心电图平线动画 + 故事文本 + 重新开始按钮

### 视觉故障特效（30 种）
随疑心度升高逐步解锁，分六类：
1. 文字效果：字体抖动、聊天框震动、幽灵文字、文字蒸发
2. 色彩扭曲：速度偏移、血脉脉冲、颜色反转、控件融化
3. 窗口效果：强地震、标题腐蚀、强制置顶
4. 血液覆盖：滴血、心电平线
5. CRT 失真：扫描线、雪花噪点、假错误弹窗、潜意识弹窗
6. Pillow 程序化纹理：血溅、暗角、色差撕裂、血液滴落、尖叫径向、地牢网格、腐蚀方块、像素融化等 12 种
7. 屠杀模式（疑心度 ≥ 85）：所有特效同时爆发 + 红黑频闪 + 80 个红色标签铺满全屏

### 第四面墙破坏
- **键盘劫持**（疑心度 ≥ 75 或逃脱率 ≥ 75）：每次按键自动输入"你走不掉的你走不掉的..."，退格键被屏蔽，伴随高频蜂鸣，只有提交被劫持文字才能解除
- **窗口缩小反逃跑**：窗口宽度 < 1000px 时全屏红色覆盖，输入框禁用
- **Alt+F4 拦截**：不关闭游戏，转而加载存档

### 心电图系统
- 45px Canvas，300 采样点 P-QRS-T 真实波形，约 33FPS 刷新
- 随疑心度变化：颜色加深、振幅增大、周期缩短
- 30 个暗红余烬粒子漂浮
- 配合 Pygame 心跳音频循环播放

### 语音合成（TTS）
- 依赖 GPT-SoVITS 本地 HTTP API（默认 127.0.0.1:9880）
- 自动端点探测（/tts、/tts_to_audio）
- 语言检测 + 模型热切换：中文→花火模型，日文→Mita 模型
- 文本清洗：移除 <think> 块、JSON、括号内容、非语言符号
- 35 秒安全超时

### 多语言系统
- 界面三语：简体中文 / English / 日本語
- 双语输出：纱希用设定语言回复，自动附加玩家语言的括号翻译
- AI 翻译自愈：检测缺失翻译时自动发起二次 API 调用补充
- 完整的三语故障特效本地化

### 存档系统
- 5 个手动槽位 + 1 个自动存档
- Ctrl+F1~F5 保存，Alt+F1~F5 加载
- 存档内容：天数、三个数值、完整聊天历史、API 配置、语言选择
- 读档完整恢复所有对话记录

### 自定义角色系统
- 配置文件 custom_characters.json
- 可自定义：名称、年龄、性格、背景故事、聊天颜色、TTS 模型权重路径
- F12 或 Ctrl+Alt+C 在纱希和自定义角色间切换
- 切换时系统提示和 TTS 权重自动更新
- 预置角色：花火（Sparkle）—— 假面愚者成员，小恶魔型病娇

### 其他机制
- 日期推进：检测睡眠/早安关键词自动推进天数
- 沉默回响：短输入有 35% 概率触发纱希引用玩家原话
- 侮辱反击：严重辱骂触发即时数值惩罚 + 强制愤怒情绪提示
- 危险词汇：触发心电图疯狂模式和窗口抖动

## 七、架构要点（重构时需关注）

1. **线程模型**：所有 tkinter 操作必须在主线程。后台线程（API 请求、TTS 合成）通过 `queue.Queue` 将结果推送给主线程的 `_process_ui_queue` 方法（每 20ms 轮询一次）
2. **状态机**：`GameState` 类持有所有可变状态，与 UI 完全解耦。UI 通过读取 GameState 属性来更新显示
3. **AI 响应格式**：`<think>...</think>` + 口语台词 + `（双语翻译）` + `||{JSON delta}||`，解析依赖正则和 JSON
4. **特效调度**：GlitchController 为桩代码——实际 30 种特效的逻辑分散在 main_window.py 中（约 500+ 行），并未完全模块化
5. **配置持久化**：`yandere_config.json` 存储 API Key 等敏感信息（明文）
6. **无外部资源依赖**：所有视觉素材（纹理、粒子）和音频（心跳）均为程序化运行时生成

## 八、总结

这是一个将 **LLM 对话生成、程序化视觉特效、语音合成、心理恐怖叙事**深度融合的实验性同人游戏。技术上以 Python + tkinter + 多线程为核心架构，通过模块化分层（6 层 / 25 个模块）实现了一定程度的解耦。但核心 UI 文件仍然偏重（~2700 行），特效逻辑分散，状态管理依赖隐式约定而非显式状态机。重构时建议关注：线程安全的 UI 框架选型、状态管理的显式化、特效系统的真正模块化、以及 AI 对话流的状态机建模。
